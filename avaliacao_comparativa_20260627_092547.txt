[Questão 3]
Pergunta: Qual a diferença entre TCP e UDP em relação à confiabilidade?
Resposta Esperada: TCP é orientado à conexão e fornece entrega confiável com controle de fluxo e congestionamento. UDP não garante entrega, sendo mais rápido e simples.
Base Model: TCP (Transmission Control Protocol) fornece uma garantia de confiabilidade: os dados são enviados com entrega confirmada, retransmissão automatica se necessário e controle de erros. UDP (User Datagram Protocol) não garante a confiabilidade; os dados são transmitidos de forma assíncrona, sem controle de erros ou retransmissões, sendo mais rápido e geralmente usado para aplicações que podem lidar com perdas de dados.
Fine-Tuned: TCP (Transmission Control Protocol) oferece confiabilidade garantida: dados são enviados com controle de erros, retransmissão automática e checksums para detectar erros. UDP (User Datagram Protocol) é não confiável; apenas o tempo real do envio/recebimento é contado, sem controle de erros ou retransmissões.

[Questão 4]
Pergunta: Em C, qual a diferença entre os operadores & e * quando usados com ponteiros?
Resposta Esperada: O operador & retorna o endereço de memória de uma variável, enquanto o operador * acessa ou altera o valor armazenado no endereço apontado.
Base Model: Os operadores '&' e '*' são fundamentais para manipulação de ponteiros em C. '&': O operador '&' retorna o endereço de memória do ponteiro (ex: int* ptr = &a; printf("%p", ptr); imprime "%p"). '*': O operador '*' é usado para desempacotar um ponteiro, retornando o valor armazenado na área de endereçamento que ele aponta (ex: int x = *ptr;).
Fine-Tuned: No contexto de ponteiros em C, os operadores '&' (atrasado) e '*' (descrente) têm funções distintas:

1. '&': Isso retorna o endereço de memória do ponteiro. Por exemplo, se você tem `int *p = &x;`, então `&p` retornaria `(int *) x`.

2. '*': Isso retorna o valor armazenado no endereço especificado pelo ponteiro. Por exemplo, se `*p = 5;`, então `*p` retornaria 5.

O que eles fazem juntos?

- Se você tiver um ponteiro `int *p;`, pode usar `p++;` para incrementar o ponteiro.
  
- Para atribuir um novo valor a uma variável através de um ponteiro, use `*(p + i) = v;`. Ex: `*(p + 0) = 42;` atribui 42 ao primeiro elemento do array.

- O operador 'dereferenciamento' é `*p;`. Ele lê o valor da variável referenciada pelo ponteiro atualmente apontado.

- No caso de arrays, como `[i]`, ele desconsidera o escopo dos sub-ponteiros, pois `p[i]` significa `*(p + i)`.

- Em loops, como `for(int i=0; i<5; i++)`, o operador '(' não está associado à função `printf()` ou `scanf()`, mas sim ao loop for, indicando o início do intervalo de iterações.

São essenciais entender seus usos separados e combinados para programação eficiente e correta em C.

Base Model: O protocolo TCP (Transmission Control Protocol) usa um handshake de três passos para estabelecer uma conexão: 1) SYN (Synchronize), onde o servidor responde com ACK; 2) SYN/ACK (Acknowledge), onde o cliente responde com ACK; 3) ACK (Acknowledgment), onde ambos os lados confirmam que o handshake foi bem-sucedido.
Fine-Tuned: O protocolo TCP (Transmission Control Protocol) usa um handshake de três passos para estabelecer uma conexão: 'syn-ack', 'establish' e 'fin'.
